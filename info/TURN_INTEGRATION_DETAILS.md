# Интеграция VK TURN Proxy в WireGuard Android

В данном документе подробно описаны изменения, внесенные в оригинальный репозиторий `wireguard-android` для поддержки проксирования трафика через TURN-серверы (VK Calls).

## Общая архитектура решения

Для обеспечения работы «в одном APK» и исключения петли трафика (цикла) внутри VPN, TURN-клиент был интегрирован непосредственно в нативную библиотеку `libwg-go.so`. 

**Ключевая особенность**: Все сокеты TURN-клиента защищаются методом `VpnService.protect(fd)`, что заставляет их трафик идти через физический сетевой интерфейс в обход создаваемого VPN-туннеля.

---

## 1. Нативный уровень (Go / C / JNI)

### `tunnel/tools/libwg-go/jni.c`
- **Стабильность JNI**: Сигнатура `wgTurnProxyStart` переведена на использование простых указателей `const char *` и типов `int` вместо структур Go и `jboolean`, что исключает ошибки выравнивания (ABI) при передаче строк.
- Добавлен механизм регистрации текущего экземпляра `VpnService` через JNI (`wgSetVpnService`).
- Реализована нативная функция `wgProtectSocket(int fd)`, которая через JNI вызывает `android.net.VpnService.protect(int fd)`.
- Экспортированы методы для управления прокси: `wgTurnProxyStart` и `wgTurnProxyStop`.

### `tunnel/tools/libwg-go/turn-client.go`
- **Sticky Stream Failover**: Реализована модель «липкого» потока. Несмотря на то, что в фоне поддерживается `n` активных DTLS-соединений, весь трафик текущей сессии WireGuard направляется через один и тот же поток. Это гарантирует стабильность UDP-порта отправителя с точки зрения сервера и исключает проблемы с Replay Protection в WireGuard. Переключение на другой поток происходит только в случае обрыва текущего.
- **Исправление маршрутизации**: Устранена критическая ошибка с сохранением адреса отправителя по указателю на переменную цикла. Теперь адрес копируется по значению, что гарантирует корректный возврат ответов в WireGuard-приложение на правильный внутренний порт.
- **Совместимость с Pion V5**: Код обновлен для работы с `pion/turn/v5` (использование поля `Conn` вместо `PacketConn`, корректная обработка контекстов).
- **DNS Fix**: Принудительно включен Go-резолвер (`GODEBUG=netdns=go`) и реализован обход локальных DNS-серверов Android (localhost), которые невозможно защитить через `protect()`. В случае обнаружения локального DNS запросы перенаправляются на Яндекс/Google DNS.
- **Безопасность API**: Все параметры запросов к VK (ссылки, токены) теперь проходят через `url.QueryEscape`.
- **Интеграция с Android**: Все сетевые вызовы (`http.Client`, `net.Dialer`) используют `Control` функцию для вызова `wgProtectSocket`.

## 2. Слой Конфигурации (Java / Module :tunnel)

### `tunnel/src/main/java/com/wireguard/config/Peer.java`
- Добавлена поддержка «расширений» конфигурации через поле `extraLines`. 
- Класс теперь сохраняет любые комментарии, начинающиеся с префикса `#@`, что позволяет хранить метаданные TURN прямо в стандартном `.conf` файле WireGuard.

### `tunnel/src/main/java/com/wireguard/config/Config.java`
- Парсер обновлен: строки, начинающиеся с `#@`, больше не игнорируются как обычные комментарии, а передаются в соответствующие секции (Interface или Peer) для обработки.

---

## 3. Модель и логика управления (Kotlin / Module :ui)

### `ui/src/main/java/com/wireguard/android/model/TunnelManager.kt`
- **Инъекция настроек**: Реализованы методы `injectTurnSettingsIntoConfig` и `extractTurnSettingsFromConfig`. Настройки TURN записываются в первый пир конфигурации в виде специальных комментариев (`#@wgt:`).
- **Стойкость данных**: При импорте туннеля из файла настройки TURN извлекаются автоматически. При сохранении — записываются и в JSON-базу, и в `.conf` файл.
- **Модификация для работы**: При включении VPN метод `modifyConfigForTurn` динамически подменяет `Endpoint` на `127.0.0.1:<localPort>`.

### `ui/src/main/java/com/wireguard/android/turn/TurnSettingsStore.kt`
- Исправлен баг «обнуления»: теперь данные сохраняются в JSON-файл всегда, даже если флаг `enabled` выключен. Файл удаляется только при полном удалении туннеля.

---

## 4. Пользовательский интерфейс

- **Детальный экран туннеля**: Добавлена карточка "TURN Proxy", которая отображает текущие параметры проксирования (Peer, VK Link, Port) сразу при выборе туннеля, если TURN включен.
- **Редактор туннеля**: 
    - Исправлено поведение полей ввода: данные не стираются при переключении чекбокса "Enable TURN proxy".
    - Добавлена валидация портов (1024-65535) и количества потоков (1-16).
- **Экспорт**: При экспорте конфигурации блок настроек TURN отделяется пустой строкой для лучшей читаемости.

---

## Формат метаданных в .conf файле

Для обеспечения совместимости настройки TURN записываются как комментарии в секции `[Peer]`:

```ini
[Peer]
PublicKey = <key>
Endpoint = vpn.example.com:51820
AllowedIPs = 0.0.0.0/0

# [Peer] TURN extensions
#@wgt:EnableTURN = true
#@wgt:UseUDP = false
#@wgt:IPPort = 1.2.3.4:56000
#@wgt:VKLink = https://vk.com/call/join/...
#@wgt:StreamNum = 4
#@wgt:LocalPort = 9000
```

## Как это работает (Workflow)

1. **Загрузка**: При старте приложения или импорте файла `TunnelManager` ищет строки `#@wgt:` в конфиге.
2. **Подключение**:
    - `TunnelManager` запускает `wgTurnProxyStart` в нативном слое.
    - Go-код параллельно устанавливает `n` DTLS-соединений через TURN-релеи.
    - Hub выбирает первый готовый поток как «активный».
    - Оригинальный `Endpoint` подменяется на `127.0.0.1:<localPort>`.
3. **Трафик**: Пакеты WireGuard инкапсулируются в выбранный поток DTLS и уходят в обход VPN-туннеля благодаря `wgProtectSocket`. При сбое потока Hub мгновенно переключается на резервный.
